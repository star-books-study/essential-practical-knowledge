# 7장. IO 병목, 어떻게 해결하지
## 네트워크 IO와 자원 효율
- 서버는 네트워크 통신을 기반으로 동작
- 네트워크를 통해 데이터를 주고 받는 과정

```java
outputStream.write(...); // 출력 스트림으로 데이터 보내기
inputStream.read(...); // 입력 스트림으로 데이터 받기
```
- 입출력(IO)를 하는 동안 스레드는 대기한다
- 입출력에 소요되는 시간은 코드를 실행하는 시간보다 훨씬 길다

> 블로킹 : 작업이 완료될 때까지 스레드가 대기하는 것

- IO 대기에 따른 CPU 낭비를 막는 법 : 스레드를 많이 만들기
- 그러나 스레드를 만드는 데도 한계가 있음 스레드는 수백 KB에서 수 MB를 사용함
- 컨텍스트 스위칭에도 시간이 걸림
- ㅅㅓ버 성능을 늘리는 다른 방법은 자원 효율을 늘리는 것
